About

This game aims to do three things.

  1. Teach you how to use your browser's developer tools.
  2. Show how easy it is to bypass client-side input validation.
  3. Show the importance of server-side input validation.

If you're viewing this page on a laptop/desktop browser, chances are, you can open your browser's developer tools from the settings menu somewhere. If you're viewing this on a mobile browser, you probably don't have access to the developer tools.

Lots of things, you can see how the html page is rendered. You can play around with the webpage and modify it to your liking. You can see how your browser communicates with the internet, and so much more! For more info on how to use the developer tools, click here.

Client-side input validation refers to validating user input while on the browser. As you can probably tell from the game, it's very easy to break it. Which is why it's crucial to implement server-side input validation and check the user input on the server. Client-side input validation is still useful to implement as it help guides the user into inputting correctly
